Introdução às Validações

As validações no Sequelize permitem definir regras que os dados devem seguir antes de serem salvos no banco de dados.

Com validações, podemos:

  • Garantir que campos obrigatórios não fiquem vazios
  • Limitar o tamanho mínimo e máximo de textos
  • Verificar formatos (e-mail, URL, etc.)
  • Definir mensagens de erro personalizadas
  • Evitar dados inconsistentes no banco de dados

As validações são executadas antes da operação SQL, ou seja, se a validação falhar, o banco de dados nem é consultado.

Estabelecendo Conexão com o Banco de Dados

const { Sequelize, Model, DataTypes } = require("sequelize");

// Abrindo conexão
const sequelize = new Sequelize({
    dialect: 'sqlite',
    storage: 'database.sqlite'
});

Conexão idêntica ao teste anterior. Importamos as classes necessárias e criamos a instância de conexão com o SQLite.

Definindo Modelo com Validações

// Definindo as classes de modelo
class Cliente extends Model {
    static init(sequelize) {
        super.init({
            nome: {
                type: DataTypes.STRING,
                validate: {
                    notEmpty: { msg: "O nome do cliente deve ser preenchido!" },
                    len: { args: [2, 50], msg: "O nome do cliente deve ter entre 2 e 50 letras!" }
                }
            }
        }, { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}

// Inicializando o modelo (CREATE TABLE)
Cliente.init(sequelize);

Neste trecho, estamos:

  1. Definindo o atributo nome com um objeto de configuração em vez de apenas o tipo
  2. Adicionando o campo type para especificar o tipo de dado (DataTypes.STRING)
  3. Adicionando o campo validate com as regras de validação:
    • notEmpty: impede que o campo seja uma string vazia ("")
    • len: define o tamanho mínimo (2) e máximo (50) de caracteres
  4. Cada validador possui uma propriedade msg com a mensagem de erro personalizada

Importante: A diferença do teste anterior é que agora o atributo nome é definido como um objeto com type e validate, em vez de apenas DataTypes.STRING.

Inserção com Sucesso (Dados Válidos)

// Inserindo um cliente válido
const cliente1 = await Cliente.create({ nome: "Alberto" });
// Cliente inserido com sucesso!

Neste trecho, estamos:

  1. Criando um cliente com nome "Alberto" (7 caracteres)
  2. O nome passa em todas as validações:
    • notEmpty: "Alberto" não é uma string vazia ✅
    • len [2, 50]: 7 está entre 2 e 50 ✅
  3. Como todas as validações passaram, o SQL INSERT é executado normalmente
✅ Validação passou!
→ "Alberto" tem 7 caracteres (entre 2 e 50)
→ "Alberto" não é vazio
→ INSERT INTO clientes (nome) VALUES ('Alberto') executado com sucesso!

Inserção com Falha (Dados Inválidos)

// Tentando inserir um cliente com nome muito curto
const cliente2 = await Cliente.create({ nome: "A" });
// SequelizeValidationError :: Problema na inserção!

Neste trecho, estamos:

  1. Tentando criar um cliente com nome "A" (apenas 1 caractere)
  2. O nome não passa na validação:
    • notEmpty: "A" não é vazio ✅
    • len [2, 50]: 1 não está entre 2 e 50 ❌
  3. Como a validação falhou, o Sequelize lança um SequelizeValidationError
  4. O SQL INSERT nunca é executado — o banco de dados é protegido

Importante: É necessário tratar esse erro com try/catch para evitar que a aplicação quebre.

❌ SequelizeValidationError
→ "O nome do cliente deve ter entre 2 e 50 letras!"
→ Nenhum SQL foi executado no banco de dados

Validadores Comuns do Sequelize

O Sequelize oferece diversos validadores prontos para uso:

Validador Descrição Exemplo
notEmpty Não permite string vazia notEmpty: { msg: "Campo obrigatório!" }
len Define tamanho mínimo e máximo len: { args: [2, 50], msg: "..." }
isEmail Verifica formato de e-mail isEmail: { msg: "E-mail inválido!" }
isNumeric Verifica se é numérico isNumeric: { msg: "Deve ser número!" }
isURL Verifica formato de URL isURL: { msg: "URL inválida!" }
isInt Verifica se é inteiro isInt: { msg: "Deve ser inteiro!" }
min / max Define valor mínimo e máximo min: { args: [0], msg: "..." }
isIn Verifica se o valor está na lista isIn: { args: [['A', 'B']], msg: "..." }
notNull Não permite valor null notNull: { msg: "Não pode ser nulo!" }

Conclusão

Neste tutorial, vimos como usar validações no Sequelize:

  • Definir validações dentro do campo validate de cada atributo do modelo
  • Diferença entre a definição simples (DataTypes.STRING) e com validação ({ type, validate })
  • Como dados válidos são inseridos normalmente
  • Como dados inválidos geram SequelizeValidationError sem afetar o banco
  • Os validadores mais comuns disponíveis

As validações são a primeira linha de defesa para garantir a integridade dos dados na sua aplicação.
