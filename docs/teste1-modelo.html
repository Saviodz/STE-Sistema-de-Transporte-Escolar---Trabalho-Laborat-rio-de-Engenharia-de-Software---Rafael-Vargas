Introdução ao Sequelize

O Sequelize é um ORM (Object-Relational Mapping) para Node.js que suporta diversos bancos de dados SQL, como PostgreSQL, MySQL, SQLite e Microsoft SQL Server.

Com o Sequelize, podemos:

  • Representar tabelas do banco de dados como classes JavaScript
  • Representar registros como instâncias dessas classes
  • Abstrair consultas SQL através de métodos JavaScript
  • Gerenciar relacionamentos entre tabelas

Estabelecendo Conexão com o Banco de Dados

const { Sequelize, Model, DataTypes } = require("sequelize");

// Abrindo conexão
const sequelize = new Sequelize({
    dialect: 'sqlite',
    storage: 'database.sqlite'
});

Neste trecho, estamos:

  1. Importando as classes necessárias do Sequelize
  2. Criando uma instância de conexão com o banco de dados SQLite
  3. Configurando o caminho do arquivo SQLite que armazenará nossos dados

O objeto sequelize será usado para todas as operações de banco de dados.

Definindo Classes de Modelo

// Definindo as classes de modelo
class Cliente extends Model {
    static init(sequelize) {
        super.init({
            nome: DataTypes.STRING
        }, { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}

// Inicializando o modelo (CREATE TABLE)
Cliente.init(sequelize);

Neste trecho, estamos:

  1. Criando uma classe Cliente que herda de Model do Sequelize
  2. Definindo um método estático init que configura os atributos da classe
  3. Especificando que a tabela terá uma coluna nome do tipo STRING
  4. Configurando o nome do modelo ('cliente') e da tabela no banco ('clientes')
  5. Inicializando o modelo, o que cria a tabela no banco de dados

O Sequelize automaticamente adiciona colunas como id, createdAt e updatedAt.

Instanciando Objetos

// Instanciando um objeto
const cliente1 = Cliente.build({ nome: "Alberto" });
console.log(cliente1 instanceof Cliente); // true
console.log(cliente1.nome); // "Alberto"

Neste trecho, estamos:

  1. Criando uma instância do modelo Cliente usando o método build()
  2. Passando um objeto com a propriedade nome como "Alberto"
  3. Verificando se a instância é do tipo Cliente (true)
  4. Acessando a propriedade nome da instância

Importante: O método build() cria apenas o objeto na memória, ainda não o salva no banco de dados.

Inserção de Dados (Primeira Maneira)

// Inserindo um objeto no banco de dados (primeira maneira)
await cliente1.save();
console.log('Alberto foi salvo no banco de dados!\n\n');

Neste trecho, estamos:

  1. Chamando o método save() na instância cliente1 que criamos anteriormente
  2. Utilizando await para aguardar a conclusão da operação assíncrona
  3. O método save() insere o objeto no banco de dados, gerando um SQL INSERT
  4. Após a operação, o objeto cliente1 terá as propriedades id, createdAt e updatedAt preenchidas

Representação da Tabela no Banco de Dados

id nome createdAt updatedAt
1 Alberto 2023-07-15 10:30:25 2023-07-15 10:30:25

Inserção de Dados (Segunda Maneira)

// Inserindo um objeto no banco de dados (segunda maneira)
const cliente2 = await Cliente.create({ nome: "Bernardo" });
const cliente3 = await Cliente.create({ nome: "Carlos" });
const cliente4 = await Cliente.create({ nome: "Daniel" });
console.log('Clientes salvos no banco de dados!\n\n');

Neste trecho, estamos:

  1. Utilizando o método estático create() da classe Cliente
  2. Este método combina os passos de build() e save() em uma única operação
  3. Criamos e salvamos três clientes diferentes em sequência
  4. O método create() retorna a instância já com as propriedades id, createdAt e updatedAt preenchidas

Diferença-chave: O método create() é mais conciso quando queremos criar e salvar em uma única operação.

Atualização de Dados

// Atualizando um objeto
cliente1.nome = "Alberto dos Santos";
await cliente1.save();
console.log('Cliente Alberto atualizado no banco de dados!\n\n');

Neste trecho, estamos:

  1. Modificando a propriedade nome do objeto cliente1 na memória
  2. Chamando o método save() para persistir a mudança no banco de dados
  3. O método save() faz um SQL UPDATE em vez de INSERT quando o objeto já existe no banco
  4. A coluna updatedAt é automaticamente atualizada com o horário da modificação

Dica: O Sequelize pode identificar quais campos foram alterados e atualizar apenas esses.

Remoção de Dados

// Deletando um objeto
await cliente2.destroy();
console.log('Cliente Bernardo (id:2) removido do banco de dados!\n\n');

Neste trecho, estamos:

  1. Chamando o método destroy() na instância cliente2 (Bernardo)
  2. Este método remove o registro correspondente do banco de dados
  3. A operação gera um SQL DELETE
  4. Após a operação, o objeto ainda existe na memória, mas não está mais vinculado ao banco

Importante: O objeto JavaScript permanece na memória, mas seu registro no banco de dados foi removido.

Consultas no Banco de Dados

findAll: Listando Todos os Registros

// findAll: listando todos
const clientes1 = await Cliente.findAll();
console.log(clientes1.every(cliente => cliente instanceof Cliente)); // true
console.log("findAll(): \n", JSON.stringify(clientes1, null, 2), "\n\n");

Neste trecho, estamos:

  1. Utilizando o método findAll() da classe Cliente para buscar todos os registros
  2. O método retorna um array de instâncias da classe Cliente
  3. Verificamos que cada elemento do array é uma instância de Cliente
  4. Convertemos o resultado para JSON para visualização

Esta consulta equivale a um SELECT * FROM clientes.

findAll: Especificando Atributos

// findAll: listando todos (especificando atributos para SELECT)
const clientes2 = await Cliente.findAll({ attributes: ['nome'] });
console.log("findAll({ attributes: ['nome'] }): \n", JSON.stringify(clientes2, null, 2), "\n\n");

Neste trecho, estamos:

  1. Utilizando o método findAll() com o parâmetro attributes
  2. Especificamos que queremos buscar apenas a coluna nome
  3. As instâncias retornadas terão apenas a propriedade nome preenchida

Esta consulta equivale a um SELECT nome FROM clientes.

findAll: Com Cláusula WHERE

// findAll: listando todos (WHERE)
const clientes3 = await Cliente.findAll({ where: { id: 3 } });
console.log("findAll({ where: {id: 3} }): \n", JSON.stringify(clientes3, null, 2), "\n\n");

Neste trecho, estamos:

  1. Utilizando o método findAll() com o parâmetro where
  2. Especificamos uma condição: apenas registros com id = 3
  3. O resultado será um array (possivelmente com apenas um elemento)

Esta consulta equivale a um SELECT * FROM clientes WHERE id = 3.

findByPk: Buscando por Chave Primária

// findByPk: listando por chave primária
const clientes4 = await Cliente.findByPk(3);
console.log("findByPk(3): \n", JSON.stringify(clientes4, null, 2), "\n\n");

Neste trecho, estamos:

  1. Utilizando o método findByPk() da classe Cliente
  2. Especificamos o valor da chave primária (3) diretamente como parâmetro
  3. O resultado será uma única instância (ou null se não encontrar)

Diferença-chave: findByPk() retorna um único objeto, não um array, e é otimizado para busca por ID.

findOne: Buscando um Único Registro

// findOne
const clientes5 = await Cliente.findOne({ where: { id: 3 } });
console.log("findOne(3): \n", JSON.stringify(clientes5, null, 2), "\n\n");

Neste trecho, estamos:

  1. Utilizando o método findOne() da classe Cliente
  2. Especificamos uma condição com where
  3. O método retorna apenas a primeira instância que satisfaz a condição
  4. Retorna null se nenhum registro for encontrado

Uso comum: findOne() é útil quando sabemos que existe apenas um registro que satisfaz a condição, ou quando queremos apenas o primeiro encontrado.

Conclusão

Neste tutorial, vimos os fundamentos do Sequelize:

  • Estabelecer conexão com o banco de dados
  • Definir modelos que representam tabelas
  • Criar e manipular instâncias de modelos
  • Persistir dados no banco (inserir, atualizar, remover)
  • Consultar dados com diferentes métodos

O Sequelize simplifica a interação com bancos de dados relacionais, permitindo trabalhar com dados utilizando objetos JavaScript em vez de SQL direto.

Para aprofundar seus conhecimentos, explore tópicos como:

  • Relacionamentos entre modelos (hasMany, belongsTo, etc.)
  • Operadores avançados para consultas
  • Transações
  • Migrações e controle de versão do banco
