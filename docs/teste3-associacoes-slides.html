🔗 O que são Associações?

As associações no Sequelize permitem estabelecer relacionamentos entre modelos — assim como em bancos de dados relacionais.

MétodoRelacionamentoExemplo
hasOne1:1 — Um para UmUsuário tem um Perfil
hasMany1:N — Um para MuitosCliente tem muitos Telefones
belongsToN:1 — Muitos para UmVenda pertence a um Cliente
belongsToManyN:N — Muitos para MuitosAluno tem muitas Disciplinas
Nesta aula, focaremos em hasMany e belongsTo — os mais comuns.

📦 Nossos Três Modelos

JavaScript 
class Cliente extends Model {
    static init(sequelize) {
        super.init({ nome: DataTypes.STRING },
        { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}
class Telefone extends Model { … numero: DataTypes.STRING … }
class Venda    extends Model { … valor: DataTypes.FLOAT  … }

⚙️ Configurando as Associações

JavaScript 
// Após init dos modelos:
Cliente.hasMany(Telefone, { as: 'telefones' });
Venda.belongsTo(Cliente);
hasMany — 1 Cliente → N Telefones
  • Cria clienteId (FK) na tabela telefones
  • Gera métodos: getTelefones(), setTelefones(), addTelefone(), removeTelefone()
belongsTo — N Vendas → 1 Cliente
  • Cria clienteId (FK) na tabela vendas
  • Gera métodos: getCliente(), setCliente()
O Sequelize cria as chaves estrangeiras automaticamente — você não precisa adicionar clienteId no modelo.

🗄️ Estrutura das Tabelas com FKs

📞 hasMany — Criando com Associação

JavaScript 
// Método 1: Criar separado e depois vincular
const cliente1  = await Cliente.create({ nome: "Alberto" });
const telefone1 = await Telefone.create({ numero: "(11) 1111-1111" });
const telefone2 = await Telefone.create({ numero: "(22) 2222-2222" });
await cliente1.setTelefones([telefone1, telefone2]);

// Método 2: Criar já com chave estrangeira
const telefone3 = await Telefone.create({
    numero: "(33) 3333-3333",
    clienteId: cliente1.id
});
Método 1: Cria os registros separadamente e depois usa setTelefones() para vincular.
Método 2: Já informa clienteId na criação — vínculo imediato.

💰 belongsTo — Criando com Associação

JavaScript 
// Método 1: Criar separado e depois vincular
const venda1 = await Venda.create({ valor: 100 });
const venda2 = await Venda.create({ valor: 200 });
await venda1.setCliente(cliente1);
await venda2.setCliente(cliente1);

// Método 2: Criar já com chave estrangeira
const venda3 = await Venda.create({
    valor: 300,
    clienteId: cliente1.id
});
Dica: Com belongsTo, cada Venda ganha o método setCliente() (singular), pois cada venda pertence a um cliente.

🔍 Consultas com include

JavaScript 
// Buscar clientes COM seus telefones (como JOINs SQL)
const clientes = await Cliente.findAll({
    include: 'telefones'
});
console.log(JSON.stringify(clientes, null, 2));
Resultado JSON 
[{
    "id": 1,
    "nome": "Alberto",
    "telefones": [
        { "id": 1, "numero": "(11) 1111-1111", "clienteId": 1 },
        { "id": 2, "numero": "(22) 2222-2222", "clienteId": 1 },
        { "id": 3, "numero": "(33) 3333-3333", "clienteId": 1 }
    ]
}]
O parâmetro include faz o Sequelize carregar os dados relacionados — sem precisar escrever JOINs SQL manualmente.

🚀 Opções Avançadas

Personalizar FK 
Cliente.hasMany(Telefone, {
    foreignKey: 'dono_id'  // Nome personalizado da FK
});
Cascade 
Cliente.hasMany(Telefone, {
    onDelete: 'CASCADE',  // Apagar cliente → apaga telefones
    onUpdate: 'CASCADE'   // Atualizar id → propaga
});
Include filtrado 
const clientes = await Cliente.findAll({
    include: [{
        model: Telefone,
        as: 'telefones',
        where: { numero: { [Op.like]: '(11)%' } }
    }]
});
✅

Conclusão

Nesta aula, aprendemos sobre associações no Sequelize:

  • hasMany — um Cliente tem muitos Telefones (1:N)
  • belongsTo — uma Venda pertence a um Cliente (N:1)
  • O Sequelize cria FKs automaticamente
  • Métodos auxiliares gerados: get, set, add, remove
  • Duas formas de vincular: métodos ou FK diretamente
  • include carrega dados relacionados (como JOINs)
Associações tornam o código mais limpo e mantêm a integridade referencial.
