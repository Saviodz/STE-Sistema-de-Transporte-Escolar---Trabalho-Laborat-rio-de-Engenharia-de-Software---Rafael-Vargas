📖 O que é o Sequelize?

O Sequelize é um ORM (Object-Relational Mapping) para Node.js que suporta diversos bancos de dados SQL, como PostgreSQL, MySQL, SQLite e SQL Server.

Com o Sequelize, podemos:
  • Representar tabelas do banco como classes JavaScript
  • Representar registros como instâncias dessas classes
  • Abstrair consultas SQL através de métodos JavaScript
  • Gerenciar relacionamentos entre tabelas

🔌 Conexão com o Banco de Dados

JavaScript 
const { Sequelize, Model, DataTypes } = require("sequelize");

// Abrindo conexão
const sequelize = new Sequelize({
    dialect: 'sqlite',
    storage: 'database.sqlite'
});
  1. Importamos as classes necessárias do Sequelize
  2. Criamos uma instância de conexão com SQLite
  3. O arquivo database.sqlite armazenará nossos dados

🏗️ Definindo Classes de Modelo

JavaScript 
// Definindo as classes de modelo
class Cliente extends Model {
    static init(sequelize) {
        super.init({
            nome: DataTypes.STRING
        }, { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}

// Inicializando o modelo (CREATE TABLE)
Cliente.init(sequelize);
O que acontece:
  1. Classe Cliente herda de Model
  2. Coluna nome do tipo STRING
  3. init() cria a tabela no banco
Colunas automáticas:
  • id — chave primária
  • createdAt — data de criação
  • updatedAt — data de atualização

🧩 Instanciando Objetos

JavaScript 
// Instanciando um objeto
const cliente1 = Cliente.build({ nome: "Alberto" });
console.log(cliente1 instanceof Cliente); // true
console.log(cliente1.nome);                 // "Alberto"
⚠️ Importante: O método build() cria o objeto apenas na memória. Ainda não foi salvo no banco de dados!

💾 Inserção — Método save()

JavaScript 
// Inserindo um objeto no banco de dados (primeira maneira)
await cliente1.save();
console.log('Alberto foi salvo no banco de dados!');
✅ Agora sim! O save() gera um SQL INSERT e persiste o objeto no banco. O id, createdAt e updatedAt são preenchidos automaticamente.

⚡ Inserção — Método create()

JavaScript 
// Inserindo um objeto no banco de dados (segunda maneira)
const cliente2 = await Cliente.create({ nome: "Bernardo" });
const cliente3 = await Cliente.create({ nome: "Carlos" });
const cliente4 = await Cliente.create({ nome: "Daniel" });
Diferença-chave: create() = build() + save() em uma única operação. Mais conciso!

✏️ Atualização de Dados

JavaScript 
// Atualizando um objeto
cliente1.nome = "Alberto dos Santos";
await cliente1.save();
console.log('Cliente Alberto atualizado!');
O save() detecta que o objeto já existe no banco e gera um UPDATE em vez de INSERT. A coluna updatedAt é atualizada automaticamente.

🗑️ Remoção de Dados

JavaScript 
// Deletando um objeto
await cliente2.destroy();
console.log('Cliente Bernardo (id:2) removido!');
⚠️ Atenção: O método destroy() gera um DELETE FROM clientes WHERE id=2. O objeto JavaScript permanece na memória, mas o registro no banco foi removido.

🔍 Consultas — findAll()

JavaScript 
// Listando todos os registros
const clientes = await Cliente.findAll();

// Especificando atributos
const nomes = await Cliente.findAll({ attributes: ['nome'] });

// Com cláusula WHERE
const filtrado = await Cliente.findAll({ where: { id: 3 } });
findAll() → SELECT * FROM clientes
Retorna um array de instâncias.
attributes → SELECT nome FROM clientes
where → SELECT * ... WHERE id = 3

🎯 Consultas — findByPk() e findOne()

findByPk 
// Busca por chave primária
const c = await Cliente.findByPk(3);
// → SELECT * ... WHERE id=3 LIMIT 1
Retorna um único objeto ou null. Otimizado para busca por ID.
findOne 
// Busca o primeiro que satisfaz
const c = await Cliente.findOne({
  where: { id: 3 }
});
// → SELECT * ... WHERE id=3 LIMIT 1
Também retorna um único objeto ou null. Aceita qualquer condição where.
Resumo: findAll() retorna array  |  findByPk() e findOne() retornam um único objeto
