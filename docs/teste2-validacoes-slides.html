Validações no Sequelize

Protegendo a integridade dos dados antes de salvar

Pressione → ou Space para começar

🛡️ Por que Validações?

As validações no Sequelize definem regras que os dados devem seguir antes de serem salvos.

✅ Com validações podemos:
  • Garantir campos obrigatórios
  • Limitar tamanho de textos
  • Verificar formatos (e-mail, URL…)
  • Definir mensagens de erro personalizadas
⚠️ Sem validações:
  • Strings vazias no banco
  • Dados inconsistentes
  • Erros difíceis de rastrear
  • Bugs em cascata
Regra: Validações são executadas antes do SQL — se falharem, o banco nunca é tocado.

🔄 Fluxo de Validação

🔀 Sem Validação vs Com Validação

Sem validação 
// Aula 1 — aceita qualquer texto
nome: DataTypes.STRING
Aceita "", "A", qualquer coisa.
Com validação 
nome: {
    type: DataTypes.STRING,
    validate: {
        notEmpty: { msg: "..." },
        len: { args: [2, 50], msg: "..." }
    }
}
Rejeita vazios e textos fora de 2–50 chars.
Diferença-chave: O atributo agora é um objeto com type + validate, em vez de só DataTypes.STRING.

📝 Modelo Completo com Validações

JavaScript 
class Cliente extends Model {
    static init(sequelize) {
        super.init({
            nome: {
                type: DataTypes.STRING,
                validate: {
                    notEmpty: {
                        msg: "O nome do cliente deve ser preenchido!"
                    },
                    len: {
                        args: [2, 50],
                        msg: "O nome deve ter entre 2 e 50 letras!"
                    }
                }
            }
        }, { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}

Cliente.init(sequelize);
Dois validadores definidos para nome: notEmpty (não pode ser "") e len (entre 2 e 50 caracteres). Cada um com msg personalizada.

✅ Inserção com Sucesso

JavaScript 
const cliente1 = await Cliente.create({ nome: "Alberto" });
// ✅ Cliente inserido com sucesso!
"Alberto" tem 7 caracteres:
  • notEmpty → "Alberto" não é vazio ✅
  • len [2, 50] → 7 está entre 2 e 50 ✅

Resultado: INSERT INTO clientes (nome) VALUES ('Alberto') executado normalmente.

❌ Inserção com Falha

JavaScript 
const cliente2 = await Cliente.create({ nome: "A" });
// ❌ SequelizeValidationError!
"A" tem apenas 1 caractere:
  • notEmpty → "A" não é vazio ✅
  • len [2, 50] → 1 não está entre 2 e 50 ❌

SequelizeValidationError: "O nome do cliente deve ter entre 2 e 50 letras!"

Nenhum SQL foi executado — o banco está protegido.

🔧 Tratamento de Erros com try/catch

JavaScript 
try {
    const cliente = await Cliente.create({ nome: "A" });
    console.log("Inserido com sucesso!");
} catch(error) {
    console.log("Problema na inserção!");
    console.log(error.message);
    // → "Validation error: O nome do cliente deve ter entre 2 e 50 letras!"
}
Sempre envolva operações de escrita em try/catch:
  • Se a validação passa → executa o bloco try normalmente
  • Se a validação falha → o catch captura o SequelizeValidationError
  • A aplicação não quebra e podemos informar o usuário

Conclusão

Neste tutorial, aprendemos sobre validações no Sequelize:

  • Definir validações com validate nos atributos do modelo
  • Diferença entre DataTypes.STRING e { type, validate }
  • Dados válidos são inseridos normalmente
  • Dados inválidos geram SequelizeValidationError — banco protegido
  • Tratar erros com try/catch
  • Validadores comuns disponíveis
Validações são a primeira linha de defesa para garantir a integridade dos dados.
