Introdução às Associações

As associações no Sequelize permitem estabelecer relacionamentos entre tabelas/modelos, assim como em bancos de dados relacionais. Os principais tipos de associações são:

  • hasOne: Um para um (1:1)
  • hasMany: Um para muitos (1:N)
  • belongsTo: Muitos para um (N:1)
  • belongsToMany: Muitos para muitos (N:N)

Neste tutorial, focaremos nos relacionamentos hasMany e belongsTo, que são os mais comuns.

Definição dos Modelos

const { Sequelize, Model, DataTypes } = require("sequelize");

const sequelize = new Sequelize({
    dialect: 'sqlite',
    storage: 'database.sqlite'
});

// Definindo as classes de modelo
class Cliente extends Model {
    static init(sequelize) {
        super.init({
            nome: DataTypes.STRING
        }, { sequelize, modelName: 'cliente', tableName: 'clientes' })
    }
}

class Telefone extends Model {
    static init(sequelize) {
        super.init({
            numero: DataTypes.STRING
        }, { sequelize, modelName: 'telefone', tableName: 'telefones' })
    }
}

class Venda extends Model {
    static init(sequelize) {
        super.init({
            valor: DataTypes.FLOAT
        }, { sequelize, modelName: 'venda', tableName: 'vendas' })
    }
}

// Inicializando os modelos
Cliente.init(sequelize);
Telefone.init(sequelize);
Venda.init(sequelize);

Neste trecho, estamos definindo três modelos que serão utilizados para demonstrar associações:

  1. Cliente: representa um cliente com um atributo nome
  2. Telefone: representa um número de telefone
  3. Venda: representa uma venda com um valor

Cada modelo é criado como uma classe que estende Model do Sequelize, e possui uma configuração básica de seus campos através do método init.

Configurando as Associações

// Configurando as associações entre os modelos
Cliente.hasMany(Telefone, {as: 'telefones'});
Venda.belongsTo(Cliente);

Este trecho configura dois tipos de associações entre nossos modelos:

Cliente.hasMany(Telefone)

Estabelece que um cliente pode ter muitos telefones (relacionamento 1:N). O parâmetro {as: 'telefones'} define o nome do atributo que será usado para acessar os telefones de um cliente.

Esta associação automaticamente adiciona uma chave estrangeira clienteId na tabela telefones.

Venda.belongsTo(Cliente)

Estabelece que uma venda pertence a um cliente (relacionamento N:1). Esta associação também adiciona uma chave estrangeira clienteId na tabela vendas.

Associação hasMany (Um para Muitos)

A associação hasMany estabelece um relacionamento um para muitos entre dois modelos. No nosso exemplo, um Cliente pode ter muitos Telefones.

Quando definimos Cliente.hasMany(Telefone, {as: 'telefones'}), o Sequelize:

  1. Adiciona uma chave estrangeira clienteId na tabela telefones
  2. Cria vários métodos de acesso no modelo Cliente:
    • cliente.getTelefones(): busca todos os telefones do cliente
    • cliente.setTelefones(telefones): define os telefones do cliente
    • cliente.addTelefone(telefone): adiciona um telefone ao cliente
    • cliente.removeTelefone(telefone): remove um telefone do cliente
    • cliente.hasTelefone(telefone): verifica se o cliente tem determinado telefone
// Inserindo um cliente e dois telefones para, depois, vincular os telefones ao cliente
const cliente1 = await Cliente.create({ nome: "Alberto" });
const telefone1 = await Telefone.create({ numero: "(11) 1111-1111" });
const telefone2 = await Telefone.create({ numero: "(22) 2222-2222" });
await cliente1.setTelefones([telefone1, telefone2]);
    
// Inserindo um telefone já com informação do cliente
const telefone3 = await Telefone.create({ numero: "(33) 3333-3333", clienteId: cliente1.id });

Associação belongsTo (Muitos para Um)

A associação belongsTo estabelece um relacionamento muitos para um entre dois modelos. No nosso exemplo, uma Venda pertence a um Cliente.

Quando definimos Venda.belongsTo(Cliente), o Sequelize:

  1. Adiciona uma chave estrangeira clienteId na tabela vendas
  2. Cria vários métodos de acesso no modelo Venda:
    • venda.getCliente(): busca o cliente associado à venda
    • venda.setCliente(cliente): define o cliente da venda

Observação: Associar com belongsTo significa que a chave estrangeira existe no modelo fonte (neste caso, em Venda).

// Inserindo duas vendas para, depois, vincular as vendas ao cliente
const venda1 = await Venda.create({ valor: 100 });
const venda2 = await Venda.create({ valor: 200 });
await venda1.setCliente(cliente1);
await venda2.setCliente(cliente1);
    
// Inserindo uma venda já com informação do cliente
const venda3 = await Venda.create({ valor: 300, clienteId: cliente1.id });

Criando Registros com Associações

Existem duas maneiras principais de criar registros com associações no Sequelize:

  1. Criação separada e posterior associação: Cria-se os registros separadamente e depois os associa usando os métodos gerados pelo Sequelize
  2. Criação com chave estrangeira direta: Insere-se o registro já informando a chave estrangeira
// Método 1: Criação separada e posterior associação
const cliente1 = await Cliente.create({ nome: "Alberto" });
const telefone1 = await Telefone.create({ numero: "(11) 1111-1111" });
await cliente1.addTelefone(telefone1);

// Método 2: Criação com chave estrangeira direta
const telefone2 = await Telefone.create({ 
    numero: "(22) 2222-2222",
    clienteId: cliente1.id 
});

Consultas Incluindo Associações

Um dos grandes benefícios de usar um ORM como o Sequelize é a capacidade de realizar consultas que automaticamente incluem dados de tabelas relacionadas, similar a JOINs em SQL.

O Sequelize fornece o parâmetro include que permite buscar dados relacionados em uma única consulta.

// Listando todos os clientes, incluindo suas informações de telefones
const clientes1 = await Cliente.findAll({include: 'telefones'});
console.log("\nCliente.findAll({include: 'telefones'}): \n", JSON.stringify(clientes1, null, 2));

Quando usamos include: 'telefones', o Sequelize:

  1. Executa uma consulta para buscar todos os clientes
  2. Para cada cliente encontrado, executa uma consulta adicional para buscar seus telefones
  3. Organiza os resultados em uma estrutura de objetos aninhados

Isso nos permite acessar facilmente as associações, como cliente.telefones, sem precisar escrever JOINs SQL manualmente.

Opções Avançadas de Associações

O Sequelize oferece várias opções avançadas ao configurar associações:

Personalização de chaves estrangeiras
Cliente.hasMany(Telefone, {
  foreignKey: 'dono_id' // Altera o nome da chave estrangeira
});
Cascata de operações
Cliente.hasMany(Telefone, {
  onDelete: 'CASCADE', // DELETE CASCADE
  onUpdate: 'CASCADE'  // UPDATE CASCADE
});

As opções de cascata definem o que acontece com os registros relacionados quando um registro principal é atualizado ou excluído.

Consultas mais complexas com include
// Opções complexas de include
const clientes = await Cliente.findAll({
  include: [{
    model: Telefone,
    as: 'telefones',
    where: { numero: { [Op.like]: '(11)%' } }
  }]
});

Essa consulta busca apenas clientes que têm telefones começando com "(11)".

Conclusão

As associações no Sequelize são poderosas e flexíveis, permitindo representar todos os tipos de relacionamentos encontrados em bancos de dados relacionais:

  • hasOne e belongsTo: para relacionamentos um-para-um
  • hasMany e belongsTo: para relacionamentos um-para-muitos
  • belongsToMany: para relacionamentos muitos-para-muitos

Principais benefícios das associações:

  • Código mais limpo e expressivo
  • Métodos auxiliares gerados automaticamente (get, set, add, etc.)
  • Facilidade para realizar consultas que incluem dados relacionados
  • Manutenção da integridade referencial (chaves estrangeiras)

Para aprofundar seus conhecimentos, explore:

  • Associações muitos-para-muitos com tabelas intermediárias
  • Eager loading (carregamento antecipado) vs. Lazy loading (carregamento preguiçoso)
  • Consultas complexas com várias associações aninhadas
  • Uso de transações para manter a consistência dos dados
